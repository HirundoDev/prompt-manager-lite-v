# Playbook v2 — DOC015: Pull Request Template

Guía para mantener y usar correctamente la plantilla de Pull Requests.

Relacionado con: `docs/DOC015-PullRequestTemplate.md`

---

## 1) Propósito

Estandarizar la información clave en PRs para revisiones eficientes y trazables.

---

## 2) Estructura obligatoria de la plantilla

1. Tipo de cambio (checklist)
2. Descripción del cambio
3. Issue relacionado (closes #)
4. Checklist de autoverificación
5. Evidencias (si aplica)
6. Notas para el revisor

Mantener comentarios guía y listas de verificación.

---

## 3) Pasos de uso

1. Crear PR utilizando esta plantilla.

2. Completar:
   - Tipo de cambio (marcar categorías correctas)
   - Descripción con motivación y contexto
   - Enlace a issue (Closes #ID)
   - Checklist (estilo, pruebas, documentación, etc.)
   - Evidencias (UI, logs) si corresponde

---

## 4) Checklist de validación

- [ ] Tipo de cambio correcto(s)
- [ ] Descripción clara
- [ ] Issue referenciado o justificación del cambio
- [ ] Checklist completado
- [ ] Pruebas pasan localmente

---

## 5) Buenas prácticas

- Pequeños PRs enfocados; evitar cambios no relacionados.
- Añadir contexto y riesgos.
- Pedir revisión a personas adecuadas.

---

## 6) Errores comunes a evitar

- PRs gigantes sin contexto.
- Falta de vínculo a issue.
- No correr pruebas ni linters antes de abrir.
